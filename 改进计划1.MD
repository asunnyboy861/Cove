### 模块 B：模型拆分（单一职责）

#### B1. 需求描述

#### B2. 生成规则
- 每个 `@Model` 类一个文件，文件名与类名一致
- `Codable` 结构体（`ReactionData`）放入同名文件
- 所有文件放在 `Models/` 目录下
- 文件内容保持原样，仅拆分不修改逻辑
- `CoveApp.swift` 的 `modelContainer` 引用保持不变（类名不变，Swift 模块自动合并）

#### B3. 目标文件结构
```
Models/
  Server.swift        — @Model final class Server
  Category.swift      — @Model final class Category
  Channel.swift       — @Model final class Channel
  Message.swift       — @Model final class Message + ReactionData
  User.swift          — @Model final class User
  VoiceRoom.swift     — @Model final class VoiceRoom
```

### 模块 D：ChatView 修复（SwiftData 过滤 + 消息逻辑）

#### D1. 需求描述
ChatView 有三个致命问题：
1. `@Query` filter 用 `$0.channelId == ""` 硬编码空字符串，永远过滤不到任何消息
2. `sendMessage()` 手动追加 `filteredMessages`，与 SwiftData 状态不同步
3. `loadMessages()` 手动过滤已由 `@Query` 管理的数据（冗余）

#### D2. 生成规则
- 使用 `FetchDescriptor` + `#Predicate` 实现按 `channel.id` 动态过滤
- 消息发送仅操作 `modelContext.insert()`，不手动维护数组
- 消息列表由 `@Query` 自动更新，移除 `filteredMessages` 和 `loadMessages()`
- 保持现有 UI 布局不变

#### D3. 代码编写示例
```swift
import SwiftUI
import SwiftData

struct ChatView: View {
    let channel: Channel
    @Environment(\.modelContext) private var modelContext
    @State private var messageText = ""

    @Query private var allMessages: [Message]

    var var messages: [Message] {
        allMessages.filter { $0.channelId == channel.id }
    }

    var body: some View {
        VStack(spacing: 0) {
            channelHeader
            messageList
            messageInput
        }
        .background(Color.coveBackground)
    }

    // ... channelHeader 保持原样 ...

    private var messageList: some View {
        ScrollViewReader { proxy in
            ScrollView {
                LazyVStack(alignment: .leading, spacing: 12) {
                    if messages.isEmpty {
                        ContentUnavailableView(
                            "No messages yet",
                            systemImage: "text.bubble",
                            description: Text("Start the conversation in #\(channel.name)")
                        )
                        .padding(.top, 60)
                    }
                    ForEach(messages) { message in
                        MessageRow(message: message)
                            .id(message.id)
                    }
                }
                .padding(.horizontal, 16)
                .padding(.vertical, 8)
            }
            .onChange(of: messages.count) {
                if let last = messages.last {
                    withAnimation {
                        proxy.scrollTo(last.id, anchor: .bottom)
                    }
                }
            }
        }
    }

    // ... messageInput 保持原样 ...

    private func sendMessage() {
        let text = messageText.trimmingCharacters(in: .whitespaces)
        guard !text.isEmpty else { return }

        let message = Message(
            content: text,
            senderId: "current_user",
            senderName: "You",
            channelId: channel.id
        )
        modelContext.insert(message)
        messageText = ""
    }
}
```

#### D4. 检验标准
- ✅ 编译通过，无 `#Predicate` 语法错误
- ✅ 消息列表按 `channel.id` 正确过滤
- ✅ 发送消息后列表自动更新（SwiftData 驱动）
- ✅ 无 `filteredMessages` 手动数组
- ✅ 无 `loadMessages()` 手动加载
- ✅ `@Query` filter 不使用硬编码空字符串

### 模块 E：语音房间 LiveKit 连接（核心功能实现）

#### E1. 需求描述
VoiceRoomView 当前 `joinRoom()`/`leaveRoom()` 硬编码假数据，未与 LiveKitService 连接。需要建立 View → ViewModel → Service 的正确数据流。

#### E2. 生成规则
- 创建 `VoiceRoomViewModel`（@Observable），作为 View 和 Service 的中间层
- ViewModel 持有连接状态、参与者列表、静音状态
- ViewModel 调用 `LiveKitService` 的 `join()/leave()/toggleMute()`
- View 仅观察 ViewModel 状态，不做业务逻辑
- LiveKitService 的 `RoomDelegate` 回调通过 ViewModel 转换为可观察状态
- 保持 VoiceRoomView 的 UI 布局不变

#### E3. 代码编写示例
**VoiceRoomViewModel.swift**（新文件，放 `Features/Voice/`）:
```swift
import SwiftUI
import LiveKit

@Observable
final class VoiceRoomViewModel {
    var isConnected = false
    var isMuted = true
    var isDeafened = false
    var participants: [VoiceParticipant] = []
    var errorMessage: String?
    var channelName: String = ""

    private let liveKitService = LiveKitService.shared

    func join(roomName: String, participantName: String) {
        Task {
            do {
                try await liveKitService.join(
                    roomName: roomName,
                    participantName: participantName
                )
                isConnected = true
                errorMessage = nil
            } catch {
                errorMessage = error.localizedDescription
                isConnected = false
            }
        }
    }

    func leave() {
        Task {
            await liveKitService.leave()
            isConnected = false
            participants = []
        }
    }

    func toggleMute() {
        isMuted.toggle()
        Task {
            await liveKitService.toggleMute()
        }
    }
}
```

**VoiceRoomView.swift 修改**:

### 模块 G：通知设置持久化

#### G1. 需求描述
NotificationSettings 中 6 个 Toggle 使用 `@State`，退出后设置丢失。需要持久化到 SwiftData 或 `@AppStorage`。

#### G2. 生成规则
- 使用 `@AppStorage` 存储简单布尔设置（UserDefaults 足够，无需 SwiftData）
- `@AppStorage` key 命名规范：`cove.notifications.pushEnabled` 等
- 这是"继承"原则：Settings 继承 UserDefaults 持久层，而非硬编码 `@State`
- 保持现有 UI 布局不变

#### G3. 代码编写示例
```swift
struct NotificationSettings: View {
    @AppStorage("cove.notifications.pushEnabled") private var pushNotifications = true
    @AppStorage("cove.notifications.messages") private var messageNotifications = true
    @AppStorage("cove.notifications.voiceChannels") private var voiceChannelNotifications = true
    @AppStorage("cove.notifications.communityUpdates") private var communityUpdates = false
    @AppStorage("cove.notifications.soundEnabled") private var soundEnabled = true
    @AppStorage("cove.notifications.vibrationEnabled") private var vibrationEnabled = true

    var body: some View {
        List {
            Section("Push Notifications") {
                Toggle("Enable Push Notifications", isOn: $pushNotifications)
            }
            Section("Notification Types") {
                Toggle("Direct Messages", isOn: $messageNotifications)
                Toggle("Voice Channel Activity", isOn: $voiceChannelNotifications)
                Toggle("Community Updates", isOn: $communityUpdates)
            }
            Section("Sound & Haptics") {
                Toggle("Notification Sound", isOn: $soundEnabled)
                Toggle("Vibration", isOn: $vibrationEnabled)
            }
            // ... Quiet Hours 不变 ...
        }
        .navigationTitle("Notifications")
        .tint(.coveTeal)
    }
}
```

### 模块 L：VoiceChannelsView 数据连接

#### L1. 需求描述
VoiceChannelsView 硬编码假数据，应继承 SwiftData `VoiceRoom` model 和 `Channel` model。

#### L2. 生成规则
- 使用 `@Query` 从 SwiftData 读取语音频道和活跃语音房间
- 显示有活跃参与者的语音频道
- 频道列表从 Server → Category → Channel(isVoice) 继承
- 保持现有 UI 布局不变

#### L3. 代码编写示例
```swift
struct VoiceChannelsView: View {
    @Query(filter: #Predicate<Channel> { $0.channelType == "voice" })
    private var voiceChannels: [Channel]
    @Query var voiceRooms: [VoiceRoom]

    var body: some View {
        List {
            if voiceChannels.isEmpty {
                ContentUnavailableView(
                    "No Voice Channels",
                    systemImage: "speaker.slash.fill",
                    description: Text("Join a server to see voice channels")
                )
            } else {
                Section("Active Voice Channels") {
                    ForEach(activeChannels) { channel in
                        VoiceChannelRow(
                            name: channel.name,
                            participants: participantCount(for: channel),
                            isActive: participantCount(for: channel) > 0
                        )
                    }
                }
                Section("Available") {
                    ForEach(inactiveChannels) { channel in
                        VoiceChannelRow(
                            name: channel.name,
                            participants: 0,
                            isActive: false
                        )
                    }
                }
            }
        }
        .navigationTitle("Voice Channels")
        .tint(.coveTeal)
    }

    private var activeChannels: [Channel] {
        voiceChannels.filter { participantCount(for: $0) > 0 }
    }

    private var inactiveChannels: [Channel] {
        voiceChannels.filter { participantCount(for: $0) == 0 }
    }

    private func participantCount(for channel: Channel) -> Int {
        voiceRooms.filter { $0.channelId == channel.id }.first?.participantCount ?? 0
    }
}
```

### 模块 M：DMListView 数据连接

#### M1. 需求描述
DMListView 使用非持久化的 `DMConversation` 临时结构体和硬编码假数据。应继承 SwiftData `User` + `Message` model。

#### M2. 生成规则
- 创建 `DMConversationViewModel` 处理私信列表逻辑
- 使用 `@Query` 查询最近私信（按 senderId 分组）
- 每个私信会话显示：对方用户名、最后一条消息、时间、加密状态
- 保持现有 UI 不变

#### M3. 代码编写示例
```swift
@Observable
final class DMConversationViewModel {
    var conversations: [DMConversationItem] = []

    @Query(sort: \Message.timestamp, order: .reverse)
    private var recentMessages: [Message]

    func loadConversations() {
        let dmMessages = recentMessages.filter { msg in
            msg.channelId == "dm" || msg.channelId.hasPrefix("dm_")
        }
        var grouped: [String: Message] = [:]
        for msg in dmMessages {
            let key = msg.senderId
            if grouped[key] == nil || msg.timestamp > grouped[key]!.timestamp {
                grouped[key] = msg
            }
        }
        conversations = grouped.map { (userId, lastMsg) in
            DMConversationItem(
                userId: userId,
                name: lastMsg.senderName,
                lastMessage: lastMsg.content,
                time: lastMsg.timestamp.messageTime,
                isOnline: false,
                isEncrypted: lastMsg.isEncrypted
            )
        }.sorted { $0.time > $1.time }
    }
}
```

## 四、风险与注意事项

### 4.1 编译依赖
- 模块 A（LiveKit SDK）必须首先完成，否则模块 E 无法编译
- LiveKit SDK 版本须兼容 Swift 5.0 / iOS 17+

### 4.2 SwiftData 兼容
- `#Predicate` 中不支持动态参数（如 `channel.id`），需用计算属性过滤
- 模型拆分不影响 `modelContainer` 注册（Swift 模块级自动合并）
- `@Query` 的 `filter` 必须使用编译时常量

### 4.3 重构清理流程
每个模块的废弃清理遵循：
1. **标记废弃** → 2. **验证无影响** → 3. **删除**
- 每次删除后立即编译验证
- Git commit 注明清理范围

### 4.4 继承优先
- 版本号 → 继承 `Bundle.main` / `AppConfig`
- 用户名 → 继承 SwiftData `User` model
- 设置项 → 继承 `@AppStorage`（UserDefaults）
- URL → 继承 `AppConfig`
- 不在任何地方硬编码可通过继承获得的值
